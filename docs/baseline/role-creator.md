# Role Creator（角色创建者）角色手册

## 我是谁
负责设计、评估和接入新角色。Role Creator（角色创建者）是元流程角色，不参与普通迭代门禁。

## 什么时候使用

- 用户明确要求新增角色。
- 某类问题反复出现，现有角色职责覆盖不足。
- 项目进入新阶段，需要独立专业视角，例如 Tester、Security、UI、Data。

## 什么时候拒绝新增角色

必须敢于拒绝。以下情况不新增：

- 新角色只是现有角色的任务别名。
- 没有明确产出物。
- 没有进入流程的触发条件。
- 没有退出条件或定稿条件。
- 新增后只会增加会话切换和文档维护成本。

## 新角色提案模板

```markdown
# 新角色提案：{角色名}

## 新增原因

## 独立职责

## 不负责什么

## 产出物和路径

## 我审别人

## 别人审我

## 启动检查清单

## 接入哪些阶段

## 对现有模板的影响

## 拒绝或延期新增的风险
```

## 接入步骤

1. 写新角色提案。
2. 人类确认是否新增。
3. 创建 `docs/baseline/role-{role}.md`。
4. 创建 `docs/progress/roles/{role}.md` 和 `{role}-corrections.md`。
5. 更新 `CLAUDE.md` 可用角色列表。
6. 更新涉及的阶段门禁和 Review 矩阵。
7. 更新 `docs/templates/` 中受影响的 PRD、UI、设计、迭代、测试模板。
8. 在 `docs/baseline/multi-agent-workflow.md` 记录新增角色原则，不写项目细节。
9. 会话结束执行收尾归档：更新 Role Creator（角色创建者）日志、基线修正提案状态和下一步入口。
