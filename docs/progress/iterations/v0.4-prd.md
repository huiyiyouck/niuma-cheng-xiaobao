## 2. 用户故事

### 2.1 UI 重构

**US-2.1.1** 作为产品所有者，我希望管理页面不再使用四个 Tab 切换，而是以频道空间为唯一视口——信息源卡片列表为主体、子频道管理为辅助抽屉，以便在一个视图中完成所有管理操作。

**US-2.1.2** 作为产品所有者，我希望系统日志从管理页的 Tab 中独立出来，作为与"浏览""管理"平级的全局顶级导航项，因为日志不隶属于任何频道空间。

**US-2.1.3** 作为产品所有者，我希望所有弹窗（删除确认、编辑、验证等）使用自定义 UI 而非浏览器原生 `alert()` / `confirm()`，以保持视觉一致性。

**US-2.1.4** 作为产品所有者，我希望看到统一的 Toast 通知（成功/错误/信息，右上角滑入，2.5 秒自动消失），替代页面内嵌的错误文字。

### 2.2 功能闭环

**US-2.2.1** 作为产品所有者，我希望能够重命名和删除频道空间。当前只能创建和列出，建错名称后无法修改。

**US-2.2.2** 作为产品所有者，我希望能够将信息源从频道解绑，而不是只能删除整个 Source。删除 Source 会导致所有频道中的绑定级联消失，风险太高。

**US-2.2.3** 作为产品所有者，我希望添加 RSS 类型的 Source 后系统能真正抓取，而不是报错"抓取器未实现"。RSS 是最基础的信息源协议。

**US-2.2.4** 作为产品所有者，我希望告警列表中的告警可标记为"已处理"，以便区分新告警和历史告警。

**US-2.2.5** 作为产品所有者，我希望生产环境配置 ADMIN_TOKEN 后，前端的写操作不会全部 403 失败。

### 2.3 功能增强

**US-2.3.1** 作为产品所有者，我希望在浏览页通过关键词搜索新闻（标题+摘要模糊匹配），以便快速找到特定内容。

**US-2.3.2** 作为产品所有者，我希望在删除 Source 时系统明确提示该 Source 已绑定了哪些频道空间，并阻止意外删除（或给出清晰的级联影响说明）。

**US-2.3.3** 作为产品所有者，我希望管理员鉴权覆盖日志查询端点（当前任何人可访问），以及有个开关可以控制是否对读取操作也要求鉴权。

**US-2.3.4** 作为产品所有者，我希望列表加载时展示骨架屏而非"加载中…"文字，以提供更好的加载体验。

**US-2.3.5** 作为产品所有者，我希望快速切换子频道筛选或排序方式时，不会因为瞬间大量请求导致页面抖动。

## 3. 功能需求

### 3.1 UI 重构（A 板块）

原型文件位置：`docs/progress/iterations/v0.4-mockups/`

| 文件 | 内容 |

#### 3.1.3 导航结构调整

当前路由结构：
```
/ → /news（浏览页）
/admin → 管理页（4 Tab：频道管理 / Source管理 / 子频道 / 日志）
```

v0.4 路由结构：
```
/ → /news（浏览页）
/admin → 管理页（统一频道空间视图 + 子频道抽屉）
/logs → 系统日志页（全局，不限频道空间）
```

App.vue 顶部导航栏对应三个链接：浏览 | 管理 | 系统日志。

#### 3.1.4 自定义 Toast 系统

- 全局单例，通过 composable（`useToast`）调用
- 支持三种类型：success（绿色）、error（红色）、info（蓝色）
- 位置：视口右上角，`position: fixed`
- 动画：`transform: translateX(120%) → translateX(0)` 滑入，2.5s 后反向滑出
- 多条 toast 堆叠显示（每条间距 8px）
- 点击 toast 可立即关闭

#### 3.1.5 自定义 Modal 系统

- 全局单例，通过 composable（`useModal`）调用
- 结构：半透明黑色遮罩层 + 居中白色卡片
- 卡片包含：标题、正文内容（slot）、取消按钮、确认按钮
- 支持键盘操作：Enter 确认、Esc 取消
- 确认按钮支持 loading 状态（如"删除中…"）
- 替换所有现有 `confirm()` 调用点：SourceManager.vue、SubChannelManager.vue 等

### 3.2 功能闭环（B1 板块）

#### 3.2.1 频道空间完整 CRUD

新增两个端点（`routes/channel-spaces.ts`）：

| 方法 | 路径 | 说明 |
|------|------|------|
| PUT | `/v1/channel-spaces/:id` | 重命名频道空间（body: `{ name, description? }`），校验 name 唯一 |
| DELETE | `/v1/channel-spaces/:id` | 删除频道空间。需二次确认：级联删除所有关联的 sub_channels、channel_sources、raw_items、processed_news、tasks、source_states |

DELETE 的前置检查：返回该空间下的数据统计供前端展示（多少条新闻、多少个 source 绑定会被删除）。

#### 3.2.2 Source 解绑端点

新增端点（`routes/bindings.ts`）：

| 方法 | 路径 | 说明 |
|------|------|------|
| DELETE | `/v1/channel-sources/:id` | 解除 Source 与频道空间的绑定。仅删除 channel_sources 行，不删除 Source 本身。同时级联删除关联的 source_states 和 tasks |

#### 3.2.3 RSS Fetcher 实现

新增 `server/src/worker/fetchers/rss.ts`，支持：

- 标准 RSS 2.0 和 Atom 格式解析
- HTTP 请求获取 XML 内容，超时 15 秒
- 提取字段映射：`<title>`、`<link>`、`<description>`、`<pubDate>` / `<published>` / `<updated>`
- `source_item_id` 使用 `<guid>` 或 `<link>`（fallback）
- 游标机制：记录最近成功抓取的文章的 `pubDate`，下次抓取时只取更新的
- 注册到 registry（`fetchers/index.ts` 增加 `import "./rss.ts"`）

非 Retryable 场景：HTTP 404（feed 不存在）、XML 解析失败（内容不是有效 RSS/Atom）

#### 3.2.4 告警已读状态

- `alerts` 表新增 `status` 字段：`VARCHAR(20) NOT NULL DEFAULT 'active'`（值：active / acknowledged / resolved）
- 新增端点：

| 方法 | 路径 | 说明 |
|------|------|------|
| PATCH | `/v1/alerts/:id` | 更新告警状态（body: `{ status }`） |
| POST | `/v1/alerts/acknowledge-all` | 批量标记所有 active 告警为 acknowledged（可选按 channel_space_id 过滤） |

- 前端告警列表展示状态标签（颜色区分），支持单条或批量标记

#### 3.2.5 前端写操作鉴权

`frontend/src/lib/http.ts` 修改：

- 从环境变量或 config 读取 admin token
- 对 POST/PUT/PATCH/DELETE 请求自动附加 `x-admin-token` header
- admin token 来源：`VITE_ADMIN_TOKEN` 环境变量（构建时注入）或浏览器 localStorage 中存储（用户在设置页输入）

同时增强后端 adminGuard：

- `/v1/admin/logs` 纳入保护范围（当前只有读操作，但属于管理功能）
- 新增配置 `ADMIN_PROTECT_READS` 控制是否对 GET 类管理端点也要求鉴权（默认 false，保持向后兼容）

#### 3.2.6 日志文件路径修复

`admin-logs.ts` 中 `LOG_FILES` 的读取路径与 winston-daily-rotate-file 输出路径对齐：

- 支持读取带日期后缀的文件（`api-YYYY-MM-DD.log`、`worker-YYYY-MM-DD.log`）
- 当天日志优先读无后缀文件，不存在时 fallback 到带日期的文件
- 读取昨天的日志时使用带日期后缀的文件

### 3.3 功能增强（B2 板块）

#### 3.3.1 浏览页关键词搜索

- 浏览页 ChannelFilter 区域新增搜索输入框
- 输入关键词后按回车触发 API 请求
- 后端新增查询参数 `q`（`GET /v1/channel-spaces/:id/news?q=...`）
- 搜索逻辑：`WHERE (pn.title ILIKE '%q%' OR pn.summary ILIKE '%q%')`
- 搜索结果支持与子频道筛选、排序组合使用

#### 3.3.2 Source 删除前绑定检查

`DELETE /v1/sources/:id` 增强：

- 删除前查询该 Source 的 channel_sources 绑定列表
- 响应中加入 `affected_bindings` 数组，列出将被级联删除的绑定信息（频道空间名、子频道名）
- 前端展示受影响绑定清单后再确认删除

#### 3.3.3 列表骨架屏

- 浏览页新闻列表加载时展示骨架屏（3 个占位卡片，脉冲动画）
- 管理页信息源列表加载时展示骨架屏（3 个占位卡片）
- 骨架屏形状与真实内容一致：标题行（短/长）、标签行、详情行

#### 3.3.4 请求防抖

- `NewsPage.vue` 中 `filterSubChannelIds` 和 `sortBy` 切换加入 300ms 防抖
- 使用 VueUse `useDebounceFn` 或手写 debounce

#### 3.3.5 管理页按原型实现

- 删除旧 `AdminPage.vue` 4-Tab 结构
- 按 `v0.4-mockups/admin.html` 原型实现新管理页
- 关键交互：
  - 频道空间行 pill 切换
  - 信息源卡片列表（类型标签、状态标签、子频道归属、抓取参数、最后抓取、连续失败数、操作按钮）
  - 内联添加表单（URL 失焦自动识别类型 → 配置 → 验证预览 → 确认）
  - 子频道管理抽屉（右侧滑出，拖拽排序，行内编辑）
  - 告警条内联展示在卡片上

#### 3.3.6 系统日志独立页

- 新建 `LogsPage.vue`
- 路由 `/logs`
- 功能与当前 `LogViewer.vue` 一致（级别/来源筛选、时间范围、关键字搜索、自动刷新、展开详情）
- 顶部说明文字"全局 · 不限频道空间"

## 6. 依赖与约束

### 6.1 外部依赖

| 依赖 | 用途 | 风险 |
|------|------|------|
| X API v2 (Basic) | Twitter 推文拉取 | 月费约 $100，频率限制 |
| OpenAI 兼容 API | LLM 翻译 + 结构化输出 | 成本随处理量增长 |
| Supabase PostgreSQL | 数据存储 | 免费版配额和休眠风险 |
| RSS feed 源 | RSS Fetcher 的数据来源 | 各 feed 服务器稳定性和格式差异 |

### 6.2 技术约束

- Node.js Fastify + TypeScript 后端（v0.3 迁移后技术栈）
- Vue 3 Composition API + TypeScript 前端
- PostgreSQL + Drizzle ORM
- Fetcher 注册表模式（registry pattern），新增 RSS 需注册到 registry
- 自定义 Toast/Modal 以 Vue composable 方式提供（`useToast`、`useModal`）
- 所有用户可见文字使用中文

### 6.3 数据库变更

本迭代仅新增一个字段：

```sql
ALTER TABLE alerts ADD COLUMN status VARCHAR(20) NOT NULL DEFAULT 'active';
-- 值：active / acknowledged / resolved
```

无需清空数据。v0.3 数据完整性保留。

## 7. 风险与缓解

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|---------|
| RSS 格式多样性导致解析失败 | 高 | 部分 RSS 源无法抓取 | 先支持 RSS 2.0 + Atom 两种最广泛的标准格式；解析失败抛出 NonRetryableError |
| 管理页重构工作量大、前端开发周期长 | 中 | 延迟 v0.4 交付 | 原型已完整确认交互，开发可 1:1 对照实现 |
| 频道空间 DELETE 级联影响数据量大 | 低 | 用户误删重要数据 | 删除前 API 返回受影响数据统计；前端二次确认 + 输入空间名称确认 |
| 前端防抖可能导致用户感知延迟 | 低 | 点击后 300ms 才刷新 | 300ms 为常用值，用户基本无感 |
| RSS Fetcher HTTP 请求可能超时或被屏蔽 | 中 | 特定 feed 源无法抓取 | 15s 超时 + User-Agent 设置 + 重试机制 |
