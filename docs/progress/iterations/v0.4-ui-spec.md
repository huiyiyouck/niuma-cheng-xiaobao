### 2.2 筛选栏（Filter Bar）

| 要求 | 说明 |
|------|------|
| 布局 | 白底卡片，`display: flex; gap: 12px; padding: 10px 16px`，与统计卡片间距 20px |
| 空间选择器 | **Pill 按钮组**（保持现有实现交互），横向排列，`border-radius: 20px`，选中：蓝底白字 `#3498db`，未选中：白底灰边。`flex-wrap: wrap` |
| 搜索输入框 | 位于空间选择器右侧，placeholder "搜索新闻…"，按回车触发 API 请求，清空时自动恢复未搜索状态。搜索期间列表显示 loading |
| 排序下拉 | 选项：最新优先 / 最高评分 / 最低评分，最小宽度 110px |
| 最低评分滑块 | 右对齐，`<input type="range">` 0-10，**步长 0.5**（匹配 `importance_score` numeric 精度），蓝色滑块 16px 圆形，选中值右侧显示"不限"或"N 分"（蓝色 12px 加粗，显示一位小数如"7.5 分"） |
| 分隔线 | `1px solid #E6E7EB` 竖线，高 20px，分隔不同筛选组 |

**防抖说明**：子频道 Pill 切换和排序下拉切换均适用 300ms 防抖（PRD 3.3.4 要求），防止快速连续操作产生请求风暴。搜索输入框按回车触发，不适用防抖。

**搜索交互细节**：
- 输入关键词后按回车触发搜索，搜索期间展示 loading（复用列表 loading）
- 搜索结果**不做**关键词高亮（v0.4 范围外）
- 清空搜索框时自动恢复未搜索状态
- 搜索条件与子频道筛选、排序取交集（组合使用）

## 5. Toast / Modal 组件规范

### 5.1 Toast 通知

| 要求 | 说明 |
|------|------|
| 调用方式 | `useToast()` composable → `toast.success(msg)` / `toast.error(msg)` / `toast.info(msg)` |
| 注入方式 | 由设计阶段决定：方案 A（`provide`/`inject`）或方案 B（模块级单例） |
| 容器位置 | `position: fixed; top: 16px; right: 16px; z-index: 100`，纵向排列 `gap: 8px` |
| 堆叠方向 | 新 toast 出现在**上方**（最新消息在最上面） |
| 单条样式 | `padding: 10px 16px; border-radius: 8px; font-size: 12px; font-weight: 600`。图标（✅/❌/ℹ️）+ 文字。max-width 360px |
| 颜色 | success：`background: #27ae60; color: #FFF`。error：`background: #e74c3c; color: #FFF`。info：`background: #1a1a2e; color: #FFF` |
| 入场动画 | `@keyframes toastIn { from { opacity: 0; transform: translateX(20px) } to { opacity: 1; transform: translateX(0) } }` 0.3s ease-out |
| 自动消失 | 2.5s 后淡出（`opacity: 0; transition: opacity 0.3s`），然后从 DOM 移除 |
| 手动关闭 | 点击任意 toast 可立即关闭 |

### 5.2 Modal 对话框

| 要求 | 说明 |
|------|------|
| 调用方式 | `useModal()` composable → `modal.confirm(title, body, { confirmText, danger })`，返回 Promise |
| 遮罩层 | `position: fixed; inset: 0; background: rgba(0,0,0,0.35); z-index: 60; display: flex; align-items: center; justify-content: center`。入场动画 `fadeIn 0.15s ease-out` |
| 对话框 | 白底，`border-radius: 12px`，`box-shadow: 0 12px 32px rgba(0,0,0,0.12)`。`padding: 24px; min-width: 360px; max-width: 440px`。入场动画 `modalIn 0.2s cubic-bezier(0.4,0,0.2,1)`（scale 0.95→1 + translateY -8px→0） |
| 标题 | 15px，font-weight 800，`color: #1a1a2e`，`margin-bottom: 8px` |
| 正文 | 13px，`color: #5F6B7A`，`line-height: 1.6`，`margin-bottom: 20px`。支持 HTML（如 `<strong>` 加粗） |
| 按钮 | 右对齐 `gap: 8px`。「取消」（白底灰边）+「确认」（primary 蓝底白字，或 danger 红字白底红边）。`padding: 8px 20px; font-size: 13px` |
| 键盘操作 | Enter 确认，Esc 取消 |
| Body scroll lock | 打开时 `document.body.style.overflow = 'hidden'`，关闭时恢复 |
| 点击遮罩层 | 关闭对话框（等于取消） |
| Loading 状态 | 确认按钮支持 loading（如"删除中…"），禁用重复点击 |

**替换范围**：所有当前使用浏览器原生 `confirm()` / `alert()` 的调用点（SourceManager.vue、SubChannelManager.vue 等），改用自定义 Modal。

### 8.1 CSS 变量迁移策略

当前 `frontend/src/style.css` 使用旧变量体系（`--primary: #2563eb`、`--bg: #ffffff` 等），v0.4 新变量体系（`--accent: #3498db`、`--bg: #FAFAFA` 等）与旧体系存在差异。推荐**渐进式迁移**——保留旧变量别名，逐步替换：

```css
:root {
  /* v0.4 新变量 */
  --bg: #FAFAFA;
  --card: #FFF;
  --accent: #3498db;
  --accent-light: #EBF5FB;
  --success: #27ae60;
  --success-light: #E8F8F0;
  --warning: #f39c12;
  --warning-light: #FEF9E7;
  --danger: #e74c3c;
  --danger-light: #FDEDEC;
  --text: #1a1a2e;
  --text-secondary: #5F6B7A;
  --text-muted: #94A3B8;
  --border: #E6E7EB;
  --border-light: #F0F1F3;

  /* 旧变量别名（兼容现有组件，渐进式淘汰） */
  --primary: var(--accent);
  --primary-2: var(--accent-light);
  --bg-page: var(--bg);
  --shadow: 0 1px 3px rgba(0,0,0,0.06);
  --radius: 12px;
}
```

迁移分两步：
1. **实现阶段**：新增/重写组件直接使用新变量；旧组件通过别名保持兼容，无需同时修改
2. **后续迭代**：统一清理旧变量别名，旧组件逐步迁移到新变量
