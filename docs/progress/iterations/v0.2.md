**#2 `_source_out_legacy` 兼容适配器无效（轻微）**

`app/routes.py:59-63`：
```python
def _source_out_legacy(r: Source) -> SourceOut:
    d = _source_out_v2(r).model_dump()
    d["name"] = r.display_name
    return SourceOut.model_validate(d)
```
意图是让旧代码通过 `source.name` 仍能访问显示名称。但 `SourceOut`（`app/schemas.py:59-68`）仅定义了 `display_name` 字段，Pydantic v2 默认忽略 extra fields——`d["name"]` 在 `model_validate` 时被静默丢弃，返回的 `SourceOut` 对象没有 `name` 属性。

好在所有前端组件已统一使用 `display_name`（`SourceManager.vue`、`BindSourceModal.vue` 等），未发现任何 `source.name` 引用。当前不会触发 bug，但 `_source_out_legacy` 与 `_source_out_v2` 功能完全等价，`d["name"] = r.display_name` 是死代码。

建议两个方向之一（可在 R2 或后续迭代处理）：
- 简单方案：删除 `_source_out_legacy`，`list_channel_sources` 直接使用 `_source_out_v2`
- 如果需要真正兼容：在 `SourceOut` 中增加 `name` 属性（computed field 或 alias），但这会污染新 Schema
