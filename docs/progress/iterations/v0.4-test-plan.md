## 测试范围

本测试计划覆盖 v0.4 三大板块（A. UI 重构 + B. 功能闭环 + C. 功能增强）的完整功能验证，对应 PRD §5 全部 22 项验收标准。

### 覆盖清单

| 板块 | 测试域 | 测试方式 |
|------|--------|---------|
| A. UI 重构 | 路由结构调整、Toast 系统、Modal 系统、管理页统一视图、系统日志独立页 | 手动测试 + 浏览器验证 |
| B1. 功能闭环 | 频道空间 CRUD（3 端点）、Source 解绑（1 端点）、子频道排序（1 端点）、RSS Fetcher、告警状态机（2 端点）、前端鉴权、日志路径修复 | API 测试 + 手动测试 |
| B2. 功能增强 | 关键词搜索、Source 删除绑定检查、骨架屏、请求防抖、管理页原型实现 | 手动测试 + API 测试 |
| 兼容性 | API 向后兼容、数据库向后兼容、前端路由兼容 | API 测试 + 手动测试 |

## 不测范围

- 单元测试覆盖（项目无测试框架，v0.4 范围外）
- 性能/压力测试（数据量 < 5 万条，性能边界已在 PRD 中评估）
- X/Twitter Fetcher（已有功能，非本迭代新增）
- 不在 v0.4 范围的功能（批量操作、CSV 导出、全文搜索等，见 PRD §4）

## 测试环境

- 后端：`server/` Fastify + TypeScript，`npm run dev`
- 前端：`frontend/` Vue 3 + Vite，`npm run dev`
- 数据库：PostgreSQL（本地或 Supabase）
- 环境变量：`DATABASE_URL`、`ADMIN_TOKEN`（生产验证）、`ADMIN_PROTECT_READS`（可选）
- 浏览器：Chrome/Firefox 最新版
