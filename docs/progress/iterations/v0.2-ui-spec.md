| 管理页 Tab 导航 | Tab 切换：频道管理 / Source 管理 / 子频道管理 / 系统日志 |

## 2. NewsPage（新闻浏览页）

| 区域 | 视觉要求 |
|------|---------|
| 统计卡片 | 4 张卡片横排，白底圆角（border-radius: 10px），阴影；数字大号加粗，颜色区分：总新闻数 #1a1a2e、今日新增 #2ecc71、活跃 Source #3498db、子频道数 #9b59b6 |
| 频道空间切换 | Pill 标签横排，选中态：深色背景 #1a1a2e 白字，未选中：白底灰色边框 |
| 子频道筛选 | 圆角 Pill（border-radius: 20px），选中态：蓝底 #e8f4fd + 蓝色边框 #3498db + 蓝色文字，未选中：白底灰色边框 |
| 排序控件 | 右对齐，下拉选择，选项：时间倒序 / 评分高→低 / 评分低→高 |
| 新闻卡片 | 白底圆角，左侧 4px 彩色竖线（评分≥8 金 #f39c12、6-8 绿 #2ecc71、<6 蓝 #3498db）；右侧圆形评分徽章（36px, border-radius: 50%），颜色与左侧竖线一致；标题 15px 加粗，摘要 13px #666 |
| 标签 Chip | 彩色半透明背景圆角 Pill（border-radius: 12-14px），11-12px 字体 |
| 空状态 | "暂无新闻，请先在管理页添加信息来源" 居中提示 |
| 详情面板 | 点击卡片后右侧滑入，宽度 400px（max-width: 92vw）；顶部 sticky header"新闻详情"+ 关闭按钮 X；背景列表不可滚动（body scroll lock）；关闭方式：遮罩层 / 关闭按钮 / Esc |
| 面板内容 | 标题 18px 加粗 → 来源+时间+评分（圆形小徽章）→ 摘要（灰底圆角块）→ 核心要点（项目符号列表）→ 标签（Chip）→ 关键实体（按类型分组）→ 原文链接 |
| 面板动画 | Transition: translateX 40px + opacity 0→1, 0.25s |

## 3. SourceManager（Source 管理）

| 区域 | 视觉要求 |
|------|---------|
| 工具栏 | 状态筛选下拉 + 类型筛选下拉 + 「+ 添加 Source」按钮（深色背景 #1a1a2e 白字） |
| Source 列表 | **表格布局**，列：名称 / URL / 类型 / 状态 / 绑定空间 / 操作 |
| 类型标签 | 彩色 Badge：X/Twitter 蓝 #e8f4fd/#3498db，RSS 橙 #fef3e2/#f39c12，HF Papers 紫 #f3e8ff/#8e44ad，未识别灰 |
| 状态标签 | 彩色圆角 Pill：已启用/已验证 绿底绿字，未验证 灰底灰字，错误 红底红字 |
| 操作列 | 文字链接，蓝色（编辑/验证/绑定），红色（删除），竖线分隔 |
| 添加弹窗 | 居中 Modal，字段：名称* / URL* / 类型（显示自动识别结果 + 可手动覆盖下拉）/ 配置 JSON 文本框；底部"取消"+"创建" |

## 4. VerifyDialog（验证弹窗）

| 区域 | 视觉要求 |
|------|---------|
| 头部 | "验证 {Source名称} — 测试抓取" |
| 成功状态 | 顶部绿色提示条"抓取成功 — 共获取 N 条" |
| 预览列表 | 每条：标题（13px 加粗）+ 时间信息，灰底圆角卡片，列表最大高度 300px 可滚动 |
| 底部按钮 | "关闭"（白底灰边）+ "标记为已验证"（绿底 #27ae60 白字） |
| Loading | 弹窗内 spinner，可关闭弹窗取消等待 |

## 5. SubChannelManager（子频道管理）

| 区域 | 视觉要求 |
|------|---------|
| 工具栏 | 频道空间下拉选择器 + 「+ 新建子频道」按钮 |
| 列表项 | 拖拽手柄（:: 图标）+ 序号 + 名称 + 绑定 Source 数 + 操作（重命名/删除） |
| 内联编辑 | 点击重命名 → 当前行变为输入框，绿色"保存"+ 灰色"取消" |
| 新建行 | 列表底部蓝色虚线边框行，输入框 + 排序号 + "添加"按钮 |

## 6. LogViewer（日志查看器）

| 区域 | 视觉要求 |
|------|---------|
| 筛选区 | **Chip 按钮组**（非下拉框）：级别用彩色 Chip（DEBUG 灰、INFO 蓝、WARNING 橙、ERROR 红），来源用 Chip（API / Worker），时间范围下拉，关键字搜索输入框，右对齐自动刷新开关 + 刷新按钮 |
| 日志行 | 等宽字体（monospace），12px；时间戳（灰）→ 级别 Badge（彩色方块，3px 圆角，白字）→ 来源标签 → 消息 |
| 级别 Badge 色 | INFO: #3498db 蓝，WARNING: #f39c12 橙，ERROR: #e74c3c 红，DEBUG: #aaa 灰 |
| 自动刷新 | 默认 10s 间隔，可开关，刷新时追加新条目不重复 |
| 展开详情 | 单条日志可点击展开，显示完整 extra 字段（JSON 格式化） |

## 7. TagChip / EntityBadge

| 组件 | 视觉要求 |
|------|---------|
| TagChip | 圆角 Pill（border-radius: 12-14px），彩色半透明背景，11-12px，可点击 |
| EntityBadge | 按类型分组，每组前有类型标签（人物/组织/产品/技术）；实体名用小方块 Badge（border-radius: 4px）：人物红 #fde8e8/#c0392b、组织蓝 #e8f4fd/#2980b9、产品绿 #e8f8e8/#27ae60、技术紫 #f3e8ff/#8e44ad |
