# 牛马程小报 · 使用说明

## 1. 系统概览

```
┌─────────────┐     ┌──────────────┐     ┌─────────────┐
│   前端页面    │ ←→  │  FastAPI 后端  │ ←→  │  PostgreSQL  │
│  Vue 3 管理台 │     │  REST + WS    │     │  (Supabase)  │
└─────────────┘     └──────┬───────┘     └─────────────┘
                           │
                    ┌──────┴───────┐
                    │    Worker     │
                    │  抓取 + AI处理 │
                    └──────────────┘
```

- **前端**：浏览器里看新闻、管理渠道
- **后端 API**：提供数据接口 + WebSocket 实时推送
- **Worker**：后台自动抓取 RSS/论文 → 调 LLM 翻译摘要 → 入库

## 2. 快速启动

### 2.1 环境准备

```bash
# 1. 安装依赖
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env，填写 DATABASE_URL 和 OPENAI_API_KEY

# 3. 创建数据库表
# 在 Supabase SQL Editor 中执行 db/schema.sql
```

### 2.2 启动服务

开三个终端分别运行：

```bash
# 终端 1：后端 API
uvicorn app.main:app --host 0.0.0.0 --port 8000

# 终端 2：Worker（抓取 + AI 处理）
python -m worker.main

# 终端 3：前端开发服务器
cd frontend && npm install && npm run dev
```

前端默认 `http://localhost:5173/`，API 文档 `http://localhost:8000/docs`。

## 3. 概念说明

### 频道空间（Channel Space）
一个逻辑隔离单元。比如你可以建 "AI 论文" 和 "科技资讯" 两个空间，各自绑定不同渠道、各自展示新闻。

### 渠道（Source）
新闻来源。目前支持五种类型：
- **rss**：任何 RSS 订阅源（arXiv、技术博客、OpenAI/DeepMind 官方博客等）
- **hf_daily_papers**：HuggingFace 每日论文 API
- **hacker_news**：Hacker News 热门 AI 讨论（Firebase API，自动过滤 AI 关键词）
- **github_trending**：GitHub Trending 开源 AI 项目（每日热榜 + API 验证）
- **semantic_scholar**：Semantic Scholar 学术论文搜索（按发布日期排序，免费 API）

### 绑定（Channel Source）
把渠道挂到某个空间下，并设置抓取策略（多久抓一次、每次抓多少条）。

## 4. 前端操作指南

### 4.1 浏览新闻

打开 `http://localhost:5173/` → 点击顶部 **"浏览"**：

- **统计卡片**：顶部 4 个数字概览（目前为前端本地计算，后续接后端统计）
- **筛选栏**：按渠道空间过滤、按评分滑块筛选、切换排序方式
- **新闻列表**：每条显示评分徽章 + 标题 + 来源 + 时间
- **点击展开**：查看完整摘要、标签、要点、原文链接
- **加载更多**：列表底部按钮分页加载
- **WS 状态灯**：绿色=实时连接中，黄色=连接中，红色=断开

### 4.2 管理渠道

点击顶部 **"管理"**：

#### 创建空间
点击 "+ 新建" → 输入名称 → 创建

#### 添加渠道
1. 先通过 API 创建 Source（见第 5 节）
2. 回到管理页，点击 **"+ 绑定新渠道"**
3. 下拉选择 Source → 设置抓取间隔和每次最大条数 → 绑定

#### 管理已有渠道
每个渠道卡片显示：
- 运行状态（绿色 "运行中" / 黄色 "暂停"）
- 抓取策略（间隔秒数、每次条数上限）

操作按钮：
- **编辑**：修改启用状态、抓取间隔、条数上限
- **暂停/启用**：暂停后不再抓取该渠道

### 4.3 查看告警

管理页底部 **"近期告警"** 区域会显示：

| 告警类型 | 含义 | 常见原因 |
|---------|------|---------|
| `fetch_failed` | 渠道连续抓取失败 | 源站挂了、网络问题、API 限流 |
| `zero_new` | 渠道 24 小时无新增内容 | 源站没更新、游标卡住了、抓取一直失败 |

告警左侧色条：黄色 = 警告，红色 = 严重。

## 5. API 初始化（命令行）

管理页目前只能管理已有渠道的绑定，**创建 Source 需要通过 API**：

### 5.1 创建 RSS 渠道

```bash
curl -X POST http://localhost:8000/v1/sources \
  -H "Content-Type: application/json" \
  -d '{"type":"rss","name":"arxiv-cs-AI","config":{"feed_url":"https://rss.arxiv.org/rss/cs.AI"}}'
```

### 5.2 创建 HF Daily Papers 渠道

```bash
curl -X POST http://localhost:8000/v1/sources \
  -H "Content-Type: application/json" \
  -d '{"type":"hf_daily_papers","name":"hf-daily-papers","config":{}}'
```

### 5.3 创建频道空间

```bash
curl -X POST http://localhost:8000/v1/channel-spaces \
  -H "Content-Type: application/json" \
  -d '{"name":"AI 论文","description":"arXiv + HF 每日论文"}'
```

### 5.4 绑定渠道到空间（替换 ID 为实际值）

```bash
curl -X POST http://localhost:8000/v1/channel-spaces/{space_id}/sources \
  -H "Content-Type: application/json" \
  -d '{"source_id":"{source_id}","enabled":true,"fetch_policy":{"schedule":{"every_seconds":600},"budget":{"max_items_per_run":20}}}'
```

以上创建好之后，回到管理页就能看到绑定了。

## 6. 环境变量说明

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `DATABASE_URL` | 必填 | PostgreSQL 连接串 |
| `OPENAI_API_KEY` | 必填 | LLM API Key（翻译+摘要用） |
| `OPENAI_BASE_URL` | `https://api.openai.com/v1` | 兼容 OpenAI 协议的 API 地址 |
| `OPENAI_MODEL` | `gpt-4o-mini` | 使用的模型 |
| `ADMIN_ALLOWED_IPS` | `127.0.0.1` | 允许调用写接口的 IP（逗号分隔） |
| `ADMIN_TOKEN` | 空 | 可选的管理 Token（请求头 `x-admin-token`） |
| `WORKER_ID` | `hostname-pid` | Worker 实例标识（启动时自动生成） |
| `SCHEDULER_SCAN_SECONDS` | `5` | 调度器扫描间隔 |
| `DEFAULT_FETCH_EVERY_SECONDS` | `600` | 默认抓取间隔（10分钟） |
| `DEFAULT_MAX_ITEMS_PER_RUN` | `20` | 每次抓取最大条数 |
| `DEFAULT_FAIL_ALERT_THRESHOLD` | `5` | 连续失败 N 次触发告警 |
| `DEFAULT_ZERO_NEW_HOURS` | `24` | N 小时无新增触发告警 |

## 7. 常见问题

### Q: Worker 启动后没有抓取？

检查：
1. 是否创建了 Source 并绑定到空间？
2. 绑定是否 `enabled: true`？
3. Worker 日志是否有报错？

### Q: 新闻列表为空？

Worker 抓取 → AI 处理需要时间。等了 1-2 分钟后点 "刷新新闻" 或看 Worker 日志确认处理进度。

### Q: "zero_new" 告警怎么办？

1. 先确认源站是否正常（浏览器打开 RSS/API 地址试试）
2. 检查 Worker 日志有没有该渠道的报错
3. 如果源站确实没更新，这个告警属于正常（只是提醒你关注）
4. 可以调大 `DEFAULT_ZERO_NEW_HOURS` 环境变量降低告警频率

### Q: 评分为什么都显示 "--"？

评分系统还没实现（占位状态）。后续会加上规则引擎 + AI 自动评分。

### Q: 前端连不上后端？

1. 检查 `frontend/.env.local`（没有就创建）：
   ```
   VITE_API_BASE_URL=http://127.0.0.1:8000
   VITE_WS_BASE_URL=ws://127.0.0.1:8000/ws
   ```
2. 确认后端已启动且端口正确
